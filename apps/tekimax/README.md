## üîê Key Features

### 1. Cryptographic Provenance (C2PA)
Every AI response is injected with an `X-C2PA-Manifest` header. This binary manifest contains:
- **Assertion of Origin**: Proof that the content was generated by a specific Sovereign Engine instance.
- **Visual Seal**: The **Tekimax Logomark** is embedded into the manifest bitstream.
- **Audit Chain**: A link back to the Sovereign CA which issued the signing credentials.

### 2. PII Vault & Scrubber
A stateless, zero-memory utility that identifies and redacts sensitive data (Email, SSN, Phone) before it reaches the LLM. It supports optional AES-256-GCM encryption for "Vaulting" PII.

### 3. Identity Provisioning
Integrated `secure_onboarding` flow that issues unique `project_id` and `api_key` credentials for granular tracking of engine usage.

## üöÄ Getting Started

### Prerequisites
- Docker & Docker Compose
- Rust (Stable)
- GitHub CLI (`gh`)

### Local Deployment
1. **Initialize the Trust Layer**:
   ```bash
   cd infrastructure/sovereign-ca
   docker-compose up -d --build
   ```

2. **Launch the Security Guard**:
   ```bash
   cd infrastructure/sovereign-guard
   docker-compose up -d
   ```

3. **Start the API Server**:
   ```bash
   cd api_server
   cargo run
   ```

4. **Run the Front-end**:
   Open `examples/neuro_demo/index.html` in any modern browser.
